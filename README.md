# Scaling Deep Learning: CNN Performance in Big Data Environments

This project evaluates how Convolutional Neural Networks (CNNs) perform when scaled across different computational environments, focusing on the interaction between deep learning models, data volume, and system infrastructure.

## Project Overview

The objective of this project was to investigate how deep learning models behave in big data contexts, with a particular focus on performance, scalability, and system constraints.

The project combines:

* A research-driven analysis of CNNs in big data systems
* Practical implementation using multiple deep learning frameworks
* Performance comparison across different computational environments

This creates a bridge between theoretical research and real-world deployment challenges.

## Key Features

* Developed CNN models using TensorFlow/Keras and PyTorch
* Trained models on benchmark datasets (CIFAR-10, Fashion-MNIST)
* Compared performance across:

  * Single-node environments
  * Distributed/big data workflows (Hadoop-based)
* Analysed how infrastructure impacts model performance
* Documented architecture design and optimisation decisions
* Delivered a working demo with step-by-step model execution

## Technologies Used

* Python
* TensorFlow / Keras
* PyTorch
* Hadoop
* Jupyter Notebook
* Virtual Machine environments

## Research Focus

This project explored key questions in modern data science:

* How do CNNs scale in big data environments?
* What are the trade-offs between single-node and distributed systems?
* How do hardware and system constraints affect deep learning performance?
* What gaps exist between research theory and practical implementation?

The work is supported by a detailed literature review, critically evaluating existing research and identifying limitations in current approaches.

## Implementation

Multiple experiments were conducted to evaluate CNN performance:

* Standard CNN architectures applied to image classification tasks
* Framework comparison (TensorFlow vs PyTorch)
* Distributed data processing using Hadoop-based workflows
* Performance evaluation based on accuracy, training time, and system limitations

The project places strong emphasis on how system design decisions influence machine learning outcomes.

## Key Insights

* Model performance depends heavily on infrastructure, not just architecture
* Distributed systems offer scalability but introduce complexity and overhead
* Hardware limitations significantly impact training efficiency
* Smaller datasets reduce the benefits of distributed processing
* There is a clear gap between theoretical scalability and practical implementation

## Limitations

* Experiments were constrained by available hardware resources
* Distributed system setup limited large-scale benchmarking
* Dataset size restricted full evaluation of scalability benefits

These limitations reflect real-world constraints and informed the evaluation of system performance.
